## マイグレーションバージョン

* 基本フォーマット
  - 作成日<YYYYMMDD>(8桁)+開発者No.(4桁)+Seq(4桁)
  - 例: 2026020100010001

* バージョン設定
  - 作成日: 2026/2/1, 開発者No. : 0001, Seq : 0001
  - マイグレーションクラスにマイグレーション属性を付与
  - 例: [FluentMigrator.Migration(2026020100010001)]

* 運用ルール
  - 基本的にはテーブル毎にマイグレーションバージョンを作成し、Seqで区別させる
  - 複数テーブルの作成・変更を一つのマイグレーションファイルに入れないようにする
  - マイグレーションバージョンはスキーマ毎に独立

## マイグレーションスキーマ

* スキーマ設定
  - マイグレーションクラスにタグ属性を付与
  -　例: [FluentMigrator.Tags("User")]

* 運用ルール
  - テーブルは必ずどこかのスキーマに所属させる
  - データ内容から適切なスキーマを選択すること
  - 適切なスキーマがない場合は、スキーマを追加することを検討
  - スキーマはMigrationSchema.csを参照
  - 例: Create.Table("UserInfo").InSchema(MigrationSchema.User).XXX;

## マイグレーションファイル, クラス名

* ファイル, クラス名
 - マイグレーションバージョン: 2026020100010001
 - 例1: _2026020100010001_Create???Master.cs
 - 例2: _2026020100010001_Alter???Master.cs
